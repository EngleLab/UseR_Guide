15.1 Aesthetics vs. Attributes

In ggplot2 we can change the looks of certain elements in our data visualization. We can create different colored data points, sizes, shapes, etc. Colloquially we might think of these as the aesthetics. However, in ggplot2 there is a distinction between aesthetics and attributes.

Aesthetics are how variables are mapped onto how something looks. Whereas, attributes are simply changing how something looks. The graphs above are examples of changing the color, size, or shape attributes of the individual data points created with geom_point().

15.2 Attributes

To set an attribute, you have to specify the argument in that specific geom layer

ggplot(iris, aes(Sepal.Length, Sepal.Width)) +
  geom_point(color = "red") +
  stat_smooth(method = "lm", se = FALSE, color = "blue", linetype = "dashed")

15.3 Aesthetics

With aesthetics we can add more variables to the graph, creating multidimensional data visualizations.

For categorical variables, color is probably the most salient and easily understood visual. Working with the iris data set used from the previous chapter let’s create the following scatterplot.

This is similar to the facet plot we created last chapter, but instead of separating species by plot, we will map Species onto a color aesthetic.

ggplot(iris, aes(Sepal.Length, Sepal.Width, color = Species)) +
  geom_point() +
  stat_smooth(method = "lm", se = FALSE)
  • Note that aesthetics can actually be set in any layer, not just in ggplot().
ggplot(iris, aes(Sepal.Length, Sepal.Width)) +
  geom_point(aes(color = Species)) +
  stat_smooth(method = "lm", se = FALSE)

However, notice how when we set aes(color = Species) in the geom_point() layer it does not apply it to the stat_smooth() layer. Therefore, we can see how setting the aesthetics in ggplot() is like setting a global aesthetic that gets passed onto to all the other layers.
